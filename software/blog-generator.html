ltlnx’s Blog Generator

I started blogging one day around 2018 on Blogger. At the time I already started writing about random computer stuff, such as Linux distros (GeeXboX) and old Windows (NT 3.51, 2000). Some time later, I read about Github Pages, moved there, and generated a blog site with Hexo. However I wanted more freedom with my pages, and eventually retracted to writing HTML, which slowed down the blogging process by a lot (aside from laziness).

Here’s where this blog generator came into play. The goal is to have enough freedom but also a degree of automation.

The script can be downloaded here.

Prerequisites

The file hierarchy it targets looks like this:

(root folder)
├── pages (destination directory)
├── src (source directory)
│   ├── desc
│   ├── blerbs (subdirectory)
│   │   ├── desc
│   │   ├── .info
│   │   ├── on-disclosure.md (post text in markdown)
│   │   └── res
│   │       └── waitinglist-grayout-mockup.png (resource files)
│   ├── footer
│   ├── header
│   ├── release-footer
│   └── release-header
├── style.css
└── u (script)

The directories and files are used as the following:

  • u is the update script. You can name it anything you want.

  • src is the source directory. It contains the markdown files and their resources.

    Details of the source directory

    In the source directory, there are:

    • header and footer are what we add in the top and bottom of all HTML pages. My header file looks like this:

      <!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8">
        <title></title>
        <meta name="viewport" content="width=device-width">
        <link rel="stylesheet" type="text/css" href="file://<root directory>/style.css">
        <link rel="alternate" type="application/rss+xml" title="All Pages on the Website" href="file://<root directory>/pages/rssfeed.xml">
    </head>
    <body>
        <div id='sbl'>
            <h3>Sidebar</h3>
            <ul>
                <li><a href='file://<root directory>/pages/index.html'>Home</a></li>
                <li><a href='file://<root directory>/pages/others/links.html'>Links</a></li>
                <li><a href='file://<root directory>/pages/blerbs/index.html'>Blerbs</a></li>
                <li><a href='file://<root directory>/pages/others/about.html'>About</a></li>
                <li><a href='file://<root directory>/pages/others/rss.html'>RSS</a></li>
                <li><a href='https://g0v.social/@ltlnx'>Mastodon</a></li>
            </ul>
        </div>
        <article>

      and the footer:

      </article>
<div id="footer">
    <p>© ltlnx under <a href="https://creativecommons.org/licenses/by-sa/4.0/">CC BY-SA 4.0</a>. Additional terms in the <a href="file://<root directory>/pages/others/about.html#copyright">About</a> page.</span>
</div>
</body>
</html>

    The <root directory> part is where the directory containing all the files is on your disk, such as /home/ltlnx/blog.

    The release-header and release-footer works the same, except that they are what we include when we are to make a release (i.e. push the content to your host of choice).

    Subdirectories inside the source directory

    In every directory in the source directory, there are the following files:

    • desc is the folder description, written in markdown. For example here’s my desc file for the blerbs directory:

      # On Blerbs

This part of the blog is inspired by Youtube channel LGR. The main
channel, LGR, posts well-researched and scripted content on various old
tech stuff among others. Meanwhile he has a division named "LGR Blerbs",
where he posts videos that are more impromptu and casual.

This blog also strives to be informative with research galore, but
occasionally I would like to write about some life events or some very
opinionated (and probably wrong) thoughts. Hence this part.

---

    • .info contains the canonical name of the directory. For example in the blerbs directory:

      FOLDERNAME="Blerbs 胡言亂語"

      It is used when we create the index pages and the navigation bar.

    • res contains the resource files used by the markdown files, such as images. They would be copied over to the destination when the script is run.

  • pages is the destination of the rendered HTML files.

  • style.css is the main CSS style specification. It would be copied into the destination folder upon a release.

Additional info

This script is intended for personal use, but I do hope that people could try it and check if it works in different settings. If you tried it and found any bugs or had any suggestions, please contact me.
