Quick and dirty Linux fixes

  2. Fedora Wifi
    • Description: Sometimes after clicking a SSID from the Wi-Fi list, it shows "Connecting..." then jumps back to disconnected state. After a quick search, it seems like the problem is caused by the DHCP client not releasing the previous IP address [citation needed]. This solution restarts the DHCP client.
    • Solution: sudo dhclient -r
    • Source:
      • From man page dhclient(8):
        -r     Release the current lease and stop the running DHCP client as previously recorded in the PID file. When shutdown via this method dhclient-script will be executed with the specific reason for calling the script set. The client normally doesn't release the current lease as this is not required by the DHCP protocol but some cable ISPs  require their clients to notify the server if they wish to release an assigned IP address.
  3. Youtube on Netsurf
    1. The main site https://youtube.com and a new aggregator https://piped.kavin.rocks won't work, use an Invidious instance (https://api.invidious.io/) instead
    2. Install mpv and youtube-dl onto your machine
    3. Copy the stream URL (should contain "watch?v=")
    4. In a terminal: (without the brackets)
      mpv (stream url)
    5. Set quality with --ytdl-format <options>
      Example:
      mpv --ytdl-format="bestvideo[height<=?""][fps<=?""]+bestaudio/best" (stream url)
      Options:
      • Resolution: [height<=?""]
        • Replace "" with how long the short side of the video should be.
        • Example: 4k (3840x2160) => [height<=?2160]
      • fps: [fps<=?""]
        • Replace "" with desired fps (30/60/120)
        • Example: 30fps => [fps<=?30]
  4. Power off external hard drive
    1. Define hard disk drive number by running lsblk
    2. udisksctl power-off -b /dev/sdX
